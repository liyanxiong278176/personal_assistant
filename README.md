# AI旅游助手 (AI Travel Assistant)

> 一款面向个人旅行者的AI智能旅游助手，通过对话式交互帮助用户规划行程、推荐景点、查询信息。同时作为校招AI应用开发岗位的面试展示项目，全面展示Agent工具调用、多模态理解、上下文记忆管理等AI应用开发能力。

[![FastAPI](https://img.shields.io/badge/FastAPI-0.115+-green?logo=fastapi)](https://fastapi.tiangolo.com/)
[![React](https://img.shields.io/badge/React-19+-blue?logo=react)](https://react.dev/)
[![Next.js](https://img.shields.io/badge/Next.js-15+-black?logo=next.js)](https://nextjs.org/)
[![License](https://img.shields.io/badge/License-MIT-yellow.svg)](LICENSE)

## 核心价值

> **智能规划 + 个性化推荐**：Agent自动调用多源API为用户生成最优旅行方案，并记住用户偏好持续优化推荐。

## 核心特性

| 特性 | 描述 | 状态 |
|------|------|------|
| **智能行程规划** | 根据目的地、时间、预算自动生成每日行程 | ✅ |
| **多Agent并行** | 复杂查询自动派生子Agent并行处理 | ✅ |
| **模型路由** | 根据复杂度自动选择deepseek-chat/deepseek-reasoner | ✅ |
| **执行计划** | Planner生成执行计划，Executor处理降级 | ✅ |
| **流式对话** | WebSocket 实时流式输出，逐字显示响应 | ✅ |
| **槽位提取** | 支持多目的地、组合简称（北上广深）、中文数字 | ✅ |
| **分层记忆** | 工作记忆、情景记忆、语义记忆三层架构 | ✅ |
| **意图分类** | 三层分类器：缓存 → 关键词 → LLM | ✅ |
| **上下文管理** | Token 估算、压缩、清理、重新注入 | ✅ |
| **工具循环** | LLM可基于工具结果继续调用，支持多轮迭代 | ✅ |
| **偏好提取** | 自动提取并保存用户偏好 | ✅ |
| **错误恢复** | 重试循环、降级响应、结构化日志 | ✅ |
| **指标收集** | Intent/Tool/Task指标统计与监控 | ✅ |

## 快速开始

### 环境要求

- **Node.js**: 18.x+
- **Python**: 3.11+
- **PostgreSQL**: 14+
- **API密钥**: DeepSeek

### 1. 克隆项目

```bash
git clone https://github.com/yourusername/travel-assistant.git
cd travel-assistant
```

### 2. 后端设置

```bash
cd backend

# 创建虚拟环境
python -m venv venv
source venv/bin/activate  # Windows: venv\Scripts\activate

# 安装依赖
pip install -r requirements.txt

# 配置环境变量
cp .env.example .env
# 编辑 .env 填入你的API密钥
```

**`.env` 配置示例：**
```bash
# 数据库
DATABASE_URL=postgresql://postgres:password@localhost:5432/travel_assistant

# DeepSeek (国产大模型，性价比高)
DEEPSEEK_API_KEY=your_deepseek_api_key
MODEL_NAME=deepseek-chat

# 高德地图
AMAP_API_KEY=your_amap_api_key

# 和风天气
QWEATHER_API_KEY=your_qweather_api_key
```

```bash
# 初始化数据库
python -c "from app.db.postgres import Database; import asyncio; asyncio.run(Database.connect())"

# 启动后端
uvicorn app.main:app --reload --host 0.0.0.0 --port 8000
```

### 3. 前端设置

```bash
cd frontend

# 安装依赖
npm install

# 配置环境变量
cp .env.local.example .env.local
```

**`.env.local` 配置示例：**
```bash
NEXT_PUBLIC_API_URL=http://localhost:8000
NEXT_PUBLIC_WS_URL=ws://localhost:8000
```

```bash
# 启动前端
npm run dev
```

### 4. 访问应用

打开浏览器访问 [http://localhost:3000](http://localhost:3000)

## API 文档

### WebSocket 端点

**连接地址**: `ws://localhost:8000/ws/chat`

**客户端消息格式**:
```json
{
  "type": "message",
  "content": "帮我规划北上广深七日游，需要住宿和天气，预算八万",
  "session_id": "session_123",
  "conversation_id": "conv_456",
  "user_id": "user_789"
}
```

**服务端响应格式**:
```json
// 流式输出
{"type": "delta", "content": "根据您的要求"}
{"type": "delta", "content": "，我为您规划了"}
...
{"type": "done", "conversation_id": "conv_456"}

// 错误
{"type": "error", "error": "处理失败"}
```

### REST API

| 端点 | 方法 | 描述 |
|------|------|------|
| `/api/agent/chat` | POST | Agent Core 聊天（非流式）|
| `/api/agent/status` | GET | Agent Core 状态 |
| `/ws/chat` | WebSocket | 流式聊天端点 |
| `/api/conversations` | POST | 创建会话 |
| `/api/conversations` | GET | 获取会话列表 |
| `/api/conversations/{id}/messages` | GET | 获取消息历史 |

完整 API 文档: [http://localhost:8000/docs](http://localhost:8000/docs)
