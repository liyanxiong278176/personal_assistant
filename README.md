# AI旅游助手 (AI Travel Assistant)

> 一款面向个人旅行者的AI智能旅游助手，通过对话式交互帮助用户规划行程、推荐景点、查询信息。

## 项目背景

### 现实问题

传统的旅行规划方式存在以下痛点：

| 痛点 | 描述 |
|------|------|
| **信息分散** | 景点信息、天气、交通、价格分散在不同平台，需要反复切换查询 |
| **规划耗时** | 制定一个合理的行程需要查阅大量攻略、对比多个方案 |
| **缺乏个性化** | 通用推荐无法考虑个人偏好（预算、兴趣、旅行风格） |
| **动态调整困难** | 行程执行中遇到天气变化或景点关闭，很难快速调整计划 |
| **记忆断层** | 每次重新规划都要重复说明偏好，无法积累个人旅行习惯 |

### 解决方案

AI旅游助手通过对话式交互，一站式解决旅行规划问题：

- **智能规划**：自动调用多源API，整合天气、地图、景点信息，生成最优路线
- **个性化推荐**：记住用户偏好（预算、兴趣、旅行风格），跨会话持续优化
- **动态调整**：实时响应变化，快速重新规划行程
- **自然交互**：像与旅行顾问对话一样简单，无需学习复杂操作

### 目标用户

1. **个人旅行者**：想要轻松规划行程，不想花时间查攻略
2. **多人出行**：需要协调不同人偏好的小团体
3. **深度游爱好者**：希望发现小众景点和地道体验
4. **时间紧张者**：需要快速得到可行的旅行方案

### 核心价值

> **"让每一次旅行都恰到好处"** — 通过AI理解和记忆，为每个人定制专属旅行方案

## 核心特性

| 特性 | 描述 |
|------|------|
| **智能行程规划** | 根据目的地、时间、预算自动生成每日行程 |
| **个性化推荐** | 记住用户偏好，跨会话持续优化推荐 |
| **多工具集成** | 天气API、地图API、POI搜索等 |
| **多模态交互** | 图片识别、图片生成、地图可视化 |
| **分层记忆** | 工作记忆、短期记忆、长期记忆三层架构 |

## 快速开始

### 环境要求

- **Node.js**: 18.x+
- **Python**: 3.11+
- **PostgreSQL**: 14+
- **API密钥**: 通义千问 (DashScope)

### 1. 克隆项目

```bash
git clone https://github.com/yourusername/travel-assistant.git
cd travel-assistant
```

### 2. 后端设置

```bash
cd backend

# 创建虚拟环境
python -m venv venv
source venv/bin/activate  # Windows: venv\Scripts\activate

# 安装依赖
pip install -r requirements.txt

# 配置环境变量
cp .env.example .env
# 编辑 .env 填入你的API密钥
```

**`.env` 配置示例：**
```bash
# 数据库
DATABASE_URL=postgresql://postgres:password@localhost:5432/travel_assistant

# 通义千问
DASHSCOPE_API_KEY=your_dashscope_api_key
MODEL_NAME=qwen-plus

# 高德地图
AMAP_API_KEY=your_amap_api_key

# 和风天气
QWEATHER_API_KEY=your_qweather_api_key
```

```bash
# 初始化数据库
python -c "from app.db.postgres import Database; import asyncio; asyncio.run(Database.connect())"

# 启动后端
uvicorn app.main:app --reload --host 0.0.0.0 --port 8000
```

### 3. 前端设置

```bash
cd frontend

# 安装依赖
npm install

# 配置环境变量
cp .env.local.example .env.local
```

**`.env.local` 配置示例：**
```bash
NEXT_PUBLIC_API_URL=http://localhost:8000
```

```bash
# 启动前端
npm run dev
```

### 4. 访问应用

打开浏览器访问 [http://localhost:3000](http://localhost:3000)

## API 文档

### WebSocket 端点

**连接地址**: `ws://localhost:8000/ws/chat`

**消息格式**:
```json
{
  "type": "message",
  "content": "你好，我想去北京旅游",
  "user_id": "user-123",
  "conversation_id": "conv-456"
}
```

**响应格式**:
```json
{
  "type": "delta",
  "content": "你好！",
  "message_id": "msg-789"
}
```

### REST API

| 端点 | 方法 | 描述 |
|------|------|------|
| `/api/conversations` | POST | 创建会话 |
| `/api/conversations` | GET | 获取会话列表 |
| `/api/conversations/{id}/messages` | GET | 获取消息历史 |
| `/api/memory/store` | POST | 存储记忆 |
| `/api/memory/context` | POST | 检索相关上下文 |

完整 API 文档: [http://localhost:8000/docs](http://localhost:8000/docs)

## 开发指南

### 添加新的工具

1. 在 `backend/app/services/` 创建新的服务文件
2. 在 `backend/app/api/chat.py` 中集成工具调用
3. 更新系统提示词以支持新工具

### 扩展记忆类型

1. 在 `backend/app/memory/prompts.py` 添加新的提取模板
2. 在 `backend/app/db/postgres.py` 添加新的数据模型
3. 更新 `preference_service.py` 以处理新类型
